# Writeups

Lead with the conclusion. Then the mechanism. Code samples last.

- First sentence states the result. The reader decides from sentence one whether to keep reading.
- Then the mechanism: how it works, in 2-4 sentences. Enough that someone could re-derive the fix without the diff.
- Code samples are evidence, not exposition. Inline a 3-5 line snippet only when prose can't carry the point.
- Assume the reader has no chat transcript. Don't write "as discussed" or "per the comparison above". Restate the thing.
- One claim per paragraph. If a paragraph has two claims, split it or pick one.
- No trailing recap. The reader has the document; summarizing it back to them is noise.
